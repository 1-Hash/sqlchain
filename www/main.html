Overview sqlChain is a compact SQL layer that runs on top of bitcoind. It extends the query options on the blockchain with a priority placed on low storage demands. This demonstration server provides multiple API (mostly compatible) interfaces:
  • Insight API
  • Blockchain.info API
  • RPC via POST, and simplified GET urls
The API page above documents the queries supported and differences to the original sites, as well as other technical details.

Two daemon programs are provided. sqlchaind updates the mysql backend from bitcoind, and sqlchain-api provides the web interface for querying the database (including this page, which is easy to extend as a blockchain web app). Using sqlChain over a pruning node, as a compact alternative to a full node, a web socket interface, and an Electrum server based on sqlChain, are currently under development.

Roll Your Own Blockchain Web App It's easy to create a web app based on sqlChain.
  1. Clone the sqlchain repo from github.
  2. Copy sqlchain-api to your own name, eg. fluffy.
  3. Edit the www directory html/js/css files to provide your web interface.
  4. Add to / remove the http handler code as needed. It shows how to query sqlchain data.
  5. Edit the cfg defaults, and/or create eg. fluffy.cfg
  6. Run ./fluffy --debug and test the site (by default on localhost:8085).
  7. Edit sqlchain.init.conf file and place in /etc/init to start both daemons at boot time.
  8. Better serve the world: run nginx as a front end and use mysql replication to scale.
